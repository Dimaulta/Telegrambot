## GolosNowBot

GolosNowBot — заготовка Telegram-бота, который будет озвучивать пересланный текст голосом. Структура каталога повторяет паттерны существующих Vapor-проектов в репозитории и служит точкой старта для дальнейшей разработки.

### Основные идеи

- Telegram-приложение на Vapor 4 с webhook-обработчиком.
- Выделенный сервис для интеграции с TTS API.
- Чёткое разделение ответственности: контроллер принимает входящие обновления Telegram, сервис формирует запросы к TTS API, модели описывают данные.

### Структура

- `Sources/App/entrypoint.swift` — точка входа приложения.
- `Sources/App/configure.swift` — конфигурация приложения, загрузка `.env`, настройка клиента.
- `Sources/App/routes.swift` — регистрация маршрутов.
- `Sources/App/Controllers/GolosNowBotController.swift` — будущая логика обработки обновлений.
- `Sources/App/Models` — модели Telegram.
- `Sources/App/Services` — сервисы для работы с TTS API.
- `docs/env.example` — список переменных окружения.

### Первоначальная настройка

1. Скопируй `docs/env.example` в `config/.env` и заполни значения токенов Telegram и TTS API.
2. Добавь URL вебхука и базовые настройки в `config/services.json`, если используешь общий конфиг.
3. Зарегистрируй новый исполняемый таргет `GolosNowBot` в корневом `Package.swift`, чтобы собирать и запускать бота.

### Ключевые переменные окружения

- `GOLOSNOWBOT_TOKEN` — токен Telegram-бота.
- `GOLOSNOWBOT_TTS_API_KEY` — ключ доступа к TTS API.
- `GOLOSNOWBOT_PORT` — порт для запуска сервиса (по умолчанию 8087).

### Следующие шаги

- Реализовать парсинг Telegram Update в моделях.
- Заполнить `GolosNowBotController` логикой приёма текста и озвучивания через TTS API.
- Дописать клиента TTS API с методами генерации аудио.
- Подготовить эндпоинты `/health` и `/webhook`, добавить обработку ошибок и логирование.

> Шаблон не содержит бизнес-логики и тестов — они появятся после уточнения TTS API и сценариев бота.
